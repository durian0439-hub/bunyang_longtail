# 블로그 글 구조·디자인·스타일 계약

## 목적

롱테일 글은 단순 정보 나열형 글이 아니라 `부동산 의사결정 체크 블로그`로 운영한다. 독자가 제목을 클릭한 뒤 첫 화면에서 자기 상황의 가능/보류/추가확인 기준과 읽는 순서를 바로 알 수 있어야 한다.

## 상단 3문단 구조

1. 검색 의도: 독자가 왜 이 글을 찾았는지 한 문장으로 짚는다.
2. 조건부 결론: 가능/보류/회피/추가확인 중 어디로 갈리는지 조건을 붙여 말한다.
3. 읽는 순서: 글 전체 목차를 반복하지 말고 독자가 먼저 볼 판단 축 3~5개만 안내한다.

예시: `읽는 순서는 자격 → 일정 → 자금 → 예외 → 공고문 최종 확인입니다.`

첫 화면 300자 안에는 기준일·금액·세대수·유찰횟수·세율·DSR/LTV 같은 숫자 또는 공식 확인처를 최소 2개 이상 넣는다. 독자가 첫 화면에서 “왜 지금 봐야 하는지”와 “무엇을 놓치면 손해인지”를 바로 이해해야 한다.

## 도메인별 판단 축

- 청약/분양: 자격, 일정, 분양가, 자금, 규제, 공고문 최종확인
- 경매: 물건 기본정보, 권리·점유, 입찰가 상한, 잔금·명도, 법원 서류 최종확인
- 세금: 발생 시점, 계산 기준, 감면·중과 예외, 신고처, 전문가 확인
- 대출: 대출 목적, DSR/LTV, 기존 대출, 실행일, 은행 상담 준비

## 디자인·이미지 배치

- 최종 사용자 노출 이미지는 GPT 생성본을 기본값으로 한다.
- 썸네일은 제목 바로 아래에 둔다.
- 본문 상단에는 원본형 인용 장식(`“` + 굵은 핵심 문장 + 얇은 구분선)을 넣어 첫 화면을 정리한다.
- 섹션 전환부에는 얇은 구분선을 사용한다. 단, 과도하게 두껍거나 광고 배너처럼 보이면 안 된다.
- 섹션 이미지가 있는 경우 이미지 바로 아래에 `〔핵심 요약〕`, `〔조건 체크〕` 같은 원본 라벨형 스티커 문구를 넣는다.
- 외부 블로그의 스티커, 유료/판매중지 스티커, 고유 장식 이미지는 그대로 복제하지 않는다. 같은 역할의 원본 도형·라벨·선으로 재작성한다.
- 섹션 설명 이미지는 해당 소제목 바로 아래에 둔다.
- 보조 이미지는 연결된 설명 텍스트보다 먼저 둔다.
- 본문 흐름은 `이미지 시각자료 → 글자료`를 유지한다.
- 로컬 fallback 이미지는 테스트와 긴급 확인용으로만 사용한다.

## 품질 게이트

- 도입부에 검색 의도, 조건부 결론, 읽는 순서가 모두 없으면 실패로 본다.
- 조건부 결론 없이 정보만 나열하면 실패로 본다.
- 첫 화면 300자 안에 숫자·기준일·공식 확인처 중 2개 이상이 없으면 실패로 본다.
- 체크리스트는 `무엇을 확인 / 어디서 확인 / 왜 필요한지`가 보여야 한다.
- FAQ는 본문 반복이 아니라 다음 단계에서 막힐 질문을 해결해야 한다.
- 투자 수익, 세금, 대출 가능성을 확정적으로 단정하지 않는다.
